🚀 テレグラム・ニュースまとめアプリ 仕様書（案）

1. プロジェクト概要TelegramにメモしたURL群をボタン一つで「未取得分のみ」抽出し、AIによる要約とカテゴリ分類を行ってリスト表示するElectronデスクトップアプリ。2. 主要機能（MVP）差分データ取得機能Telegramの特定チャットからメッセージをスキャン。メッセージ内のURLを抽出。データベース（SQLite）を照合し、過去に処理していない「新しいURL」のみをピックアップ。Webスクレイピング & 要約機能抽出したURLにアクセスし、記事本文を抽出。Gemini APIを利用し、「3行要約」と「記事のジャンル（トレンド）」を生成。ダッシュボード（リスト表示）要約された記事をカード形式で一覧表示。記事の元リンクへのジャンプ機能。「重要」「後で読む」などのステータス管理（任意）。3. システムアーキテクチャアプリは以下の3つのコンポーネントで構成されます。Frontend (Renderer): React / TypeScript (UI表示、更新ボタン、リスト閲覧)Backend (Main): Node.js (Telegram API通信、スクレイピング、DB操作、Gemini API通信)Database: SQLite (処理済みURLの保存、要約結果のキャッシュ)4. 技術スタックカテゴリ選定技術理由フレームワークElectronデスクトップアプリとして配布・利用するため。UIライブラリReact + Tailwind CSS高速なUI開発と、モダンなカードデザインのため。Telegram連携TelegrafBot APIを利用。扱いが非常に簡単。スクレイピング@mozilla/readability記事の「本文」だけを抽出するのに最適。AI (要約)Gemini API無料枠が大きく、長文入力に強いため。DBbetter-sqlite3Node.jsで動作する高速なローカルDB。5. データ構造（DB設計案）処理済みデータを管理するために、以下のテーブルを作成します。SQLCREATE TABLE articles (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  url TEXT UNIQUE,               -- 重複取得防止用
  title TEXT,                    -- 記事タイトル
  content_summary TEXT,          -- AIによる要約
  category TEXT,                 -- AIによるジャンル分け
  source_date DATETIME,          -- Telegramで投稿された日時
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
6. 処理フロー（ユーザーが「更新」ボタンを押した時）Telegram Fetch: Botがメッセージ履歴を遡り、URLを含むメッセージを取得。Filtering: DB内のurlと照合し、未登録のURLのみをリスト化。Crawling: 各URLの本文を取得。AI Processing:Prompt: 以下の記事を3行で要約し、1つのカテゴリ（例：テクノロジー、政治、経済）を決定してJSON形式で返してください。Save & Display: 結果をDBに保存し、UI上のリストを更新。7. UIイメージ案サイドバー: 「未読」「アーカイブ」「設定（APIキー入力）」メインエリア: * 上部に「最新の記事を取得（更新ボタン）」その下にカード型のリスト（タイトル、要約、タグ、日付）💡 開発へのアドバイス最初の関門：Telegram Botの権限Botをグループに入れてメモを送る場合、Botがメッセージを読み取れるように「Privacy Mode」を Disable に設定するか、Botとの1対1のチャットでメモを送るようにしてください（BotFatherから設定可能です）。AI要約のコツGemini APIに渡す際、単に「要約して」と言うよりも、「箇条書きで3点」「専門用語を噛み砕いて」などの条件（システムプロンプト）を指定すると、リスト形式の画面で見やすくなります。この仕様書で、まずは「どの部分から（例：UIから、あるいはTelegram連携から）」作り始めたいですか？具体的なコードの書き出しが必要であれば、特定の部分を深掘りしてお伝えします。
